郑玄《三礼注》直训的提取和系联分析

作业一：运用正则表达式提取郑玄三礼注直训材料

预计实现目标

运用网络爬虫技术获得网络语料，并通过正则表达式提取郑玄《三礼注》直训材料，为进一步研究提供基础材料。

操作步骤

1、从识典古籍中爬取郑玄三礼注全文。以《礼记》为例：

  • 访问识典古籍，找到对应阅读页面，展开所有折叠层，显示所有章节信息，并将网页保存到本地。
  • 通过正则表达式，提取出每一章节的标题和链接。页面中的章节有多层层次结构，所以要提取出最小单位的章节作为爬取目标，否则会重复爬取。本文采用先匹配，再去除上层结构的方法，提取出较为规整的标题和链接。
  • 通过 request 库，依次访问章节链接。通过正则表达式，匹配到正文对应的
    标签内容。
  • 以此类推，爬取《仪礼》《周礼》

2、清洗爬取的数据，提取出成组的经文和注文。

因为本文关注的是郑玄的训诂系统，所以要筛选出郑玄作注的部分。为了后续阅读溯源的方便，应该存储为表格文件，对应篇章、经文、注文。故此本文使用正则表达式和 pandas 库，对初始数据进行格式化处理。

  • 遍历 html 文件，通过正则表达式，找到其中 style 为 line-normal 及 line-comment 的标签内容，存储到列表中。
  • 处理爬取内容的无效分段，运用条件分支语句，合并相邻的同为经文或同为注文的内容。
  • 对注文进行处理，分析文本可得”○” 后内容是陆德明音义，剔除这一部分。同时去除经文中的特殊符号”○”。
  • 存储每条数据的经文和注文，保存至 csv 文件中。

3、考察三礼注注释情况，编写正则表达式匹配直训材料。

  • pandas 库读取上一步存储的 csv 文件。
  • 遍历注文，找到有对应训诂体式，训诂用语的经注组。
  • 将匹配到的经注组存储到新的 csv 文件中。
  • 除了匹配直训材料，某读为某，某读若某，某当为某等训诂用语用例也可同理得到。

4、查看匹配结果，进行人工筛选处理

发现之前正则表达式无法匹配或错误匹配的情况，重新编写正则表达式，并进行人工补充和整理。

  • pat = re.compile("^((.)，(.)也)") 直训在句首的情况
  • pat = re.compile("。((.)，(.)也)") 直训接在一句后的情况
  • pat = re.compile("((.)，.也，(.)也)") 两个直训相连，识别后一个直训。

结果分析

本文提取出了特定格式的直训注条及其注字与被注字，其基本格式为” 某，某也。“，重叠格式为” 某，某也，某也。“同时，通过改变正则表达式，也能提取出包含其他训诂用语的注条，如” 某当为某 “” 某读为某 “” 某读若某 “等。此方法能够提取特定格式的注条，同时提取出其正文，能够为进一步研究提供方便的材料。

其局限性体现在，不能同人工一样识别各种训释注条的变例，需要人工进一步增补和调整。如 “釋，猶去也。”“犹” 字连接注字与被注字，这种关系类似直训但不同于直训，有时是义隔而通或文意训释，需要人工判断，暂时没有放入检索结果中。又如一些隐藏在句意串讲中的字义解释，难以用统一的格式提取。此外，由于直训本身的代语机制，构成直训的两个词之间可能并非词义训释，所得到的结果仍然需要人工离析。

作业二：运用 gephi 生成郑玄《三礼注》直训系联图

预计实现目标

生成郑玄《三礼注》直训材料的网络关系图。以训释词为出点，以被训释词为入点，定义有向网络关系图。基于网络关系图，分析训释词和被训释词的关系。其中，特别关注有较大度和较大连出度的节点的特点，并且找到三礼注经常构成训释关系的训释词和被训释词。

操作步骤

1、导入数据

  • 根据作业一得到的直训注释字和被注释字，得到郑玄《三礼注》的有向向量表，导入 gephi 生成网络图。
  • 复制 id 列至 label 列

2、修改网络图外观

  • 节点颜色：度，由黄（小）到蓝（大）
  • 节点大小：连出度，由小到大
  • 边具备节点颜色
  • 显示标签，调节字体字号
  • 调节整体布局，预览导出

结果分析及结论

例图，详见 pdf 及 gephi 文件

关注节点：在本文的图示中，网络图中较大的节点，代表这个词连出度相对较高，即这个词经常作为注释词去训释其他词。网络中颜色较深的节点，代表这个词的度相对较高，即这个词经常出现在直训注条中，其中包括了作为注字和作为被注字的情况。颜色深、节点大的词，它们经常作为训释词去解释其他词，是语言中的基本词汇，往往具有常用性、单一性和广义性。颜色深、节点小的词，它们可以被多个训释词训释，却很少担任训释词的角色，说明其具有多义性，在多个义项上被训释，又因其多义性不适于训释其他词。颜色浅，节点小的词，为出现在直训词条中出现的频次较少的词，一方面可能为生僻词，另一方面可能是并不经常出注的常用词，只是在特定的语境下的文意训释偶然出现。

关注边：本文图示中的边，连接两个节点，代表着这两个词之间构成了训释关系。边的方向由训释词指向被训释词，颜色深的词通常是多条边的发出者，颜色浅的词通常是边的接受者，并不常充当训释词。边的粗细表示这一组训释出现的频次，较粗的边则代表此组训释常常出现，为三礼常训，郑玄不烦重复训释，证明其在语境中可能引起歧义，需要每每加注以说明，或者是经典常用但语义已经不为人所熟知，需要特加说明。这些情况都是值得进一步考察的。较细的边则证明此训出现次数较少，一方面，可能是在相似的语境下，郑玄不再重复出注，为注家精简注释的做法；另一方面，也可能是该字生僻，在经文中出现较少。两种情况都符合训释的基本原则。

借由计算机技术，能够实现人工难以实现的大规模计算，对于直训关系进行穷尽性的系联。通过节点大小的分析，可以归纳出训释系联的焦点词，由焦点词作为辐射点，沿着其聚合关系辐射开去，形成无数个以该词汇成分为核心的聚合网络，直观地呈现三礼直训的词汇系联体系。通过对边及边的方向的观察，又能够发现诸如互训、递训、语义场等之前需要大量人工或者人工难以发现的语言现象。这种清晰高效的系联方式，给予汉语言文字学研究极大的便利。
